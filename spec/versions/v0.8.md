# v0.8 — 动态规划：根据学情调整学习计划

> 目标：Planner 不再只被调用一次，Assistant 可根据学情决定 `replan`，触发 Planner 二次介入，动态增删改剩余步骤。

## 架构变更

```
v0.7                                   v0.8
────                                   ────
planner(一次) → assistant → ...        planner → assistant → resource → practice → diagnose → assistant
                                              ↑                                         │
                                              └────────── replan ──────────────────────┘
                                                        next/repeat → resource
                                                        done → END
```

- **Planner** 改为双模式：初始规划（无 plan 时）/ 动态重规划（有 plan + feedback 时）
- **Assistant** 路由增加 `replan` 选项，由 LLM 判断是否需要调整计划
- **图结构**新增 `assistant → planner` 条件边
- **前端**计划页支持动态更新

## v0.8-1 Planner 升级：支持动态重规划 [ ]

### 目标
Planner 从「只做初始规划」升级为双模式：首次调用创建计划，replan 调用读取全局状态动态增删改步骤。

### 依赖
- 无依赖

### 改动文件

| 操作 | 文件 | 说明 |
|------|------|------|
| 修改 | `server/orchestrator/agents/planner.py` | 双模式逻辑 |
| 修改 | `data/prompts/planner.md` | 双模式 Prompt |

### 实现

**planner.py** — 双模式：

```python
def planner_node(state: AgentState) -> dict:
    plan = state.get("plan", [])
    
    if not plan:
        # 模式 1：初始规划
        return _initial_plan(state)
    else:
        # 模式 2：动态重规划
        return _replan(state)


def _initial_plan(state: AgentState) -> dict:
    prompt = render_prompt("planner", task_goal=state["task_goal"], mode="initial")
    result = llm_invoke_json(prompt)
    return {"plan": result["plan"]}


def _replan(state: AgentState) -> dict:
    """根据学情调整计划。

    输入：当前 plan + step_history + feedback + current_step
    输出：调整后的 plan（可增删改步骤）
    规则：
    - 已完成步骤（step_index < current_step）保持不变
    - 仅调整未开始的步骤（step_index >= current_step）
    - 当前步骤如卡住多轮可拆分为小步骤
    """
    step_index = state["current_step"]
    future_plan = state["plan"][step_index:]  # 当前 + 剩余步骤
    completed_plan = state["plan"][:step_index]  # 已完成步骤

    prompt = render_prompt(
        "planner",
        task_goal=state["task_goal"],
        mode="replan",
        completed_steps=json.dumps(completed_plan, ensure_ascii=False),
        remaining_steps=json.dumps(future_plan, ensure_ascii=False),
        current_step=step_index + 1,
        step_history=json.dumps(state.get("step_history", []), ensure_ascii=False),
        feedback=json.dumps(state.get("feedback", {}), ensure_ascii=False),
    )
    result = llm_invoke_json(prompt)
    # 输出: {"plan": [...]} — 完整的 已完成 + 重规划的剩余 步骤
    new_future = result["plan"]
    return {"plan": completed_plan + new_future, "current_step": step_index}
```

**planner.md** — 双模式 Prompt：

```
# Role
你是一位经验丰富的高一数学教师，擅长制定和调整学习计划。

# Mode
当前模式：{mode}

---

## 初始规划模式 (initial)

### Task
根据学生的学习目标，制定一份 2-5 步的学习计划。

### Input
- 学习目标：{task_goal}

### Output Format
严格输出 JSON：
```json
{{
  "plan": [
    {{"title": "步骤名（10字内）", "desc": "具体学习内容和标准（40字内）"}}
  ]
}}
```

### Constraints
- 2-5 个步骤，由浅入深：概念 → 方法 → 应用
- 每个步骤有明确的学习产出标准

---

## 重规划模式 (replan)

### Task
学生已完成部分步骤，当前在某一步遇到困难或表现优异，需要调整剩余学习计划。

### Input
- 学习总目标：{task_goal}
- 当前在第 {current_step} 步
- 已完成步骤：
{completed_steps}
- 剩余步骤（含当前步）：
{remaining_steps}
- 学习历史：
{step_history}
- 诊断反馈：
{feedback}

### 调整策略
1. **步骤卡住**（同一步多轮正确率低）→ 拆分为 2-3 个更细的小步骤
2. **基础薄弱**（feedback.weaknesses 暴露前置知识欠缺）→ 插入前置补救步骤
3. **掌握迅速**（连续多步全对）→ 合并后续相似步骤，减少冗余
4. **正常推进**→ 保持剩余步骤不变

### Output Format
严格输出 JSON，plan 只包含**当前及剩余步骤**（不含已完成的）：
```json
{{
  "plan": [
    {{"title": "...", "desc": "..."}}
  ]
}}
```

### Constraints
- 当前步骤为 plan[0]，只调整它及之后的步骤
- 2-5 个步骤（当前 + 剩余）
- 如果无需调整，原样返回剩余步骤
```

### 测试
- 调真实 LLM：初始模式返回 2-5 步
- 调真实 LLM：replan 模式传入卡住上下文，返回更新后的步骤

### 验收标准
- [ ] 初始规划（plan=[]）→ 返回 2-5 个步骤
- [ ] 重规划（有 plan + feedback + 多轮卡住）→ 返回更新后的步骤
- [ ] 已完成步骤不被修改

## v0.8-2 Assistant 升级 + 图结构调整 [ ]

### 目标
Assistant 路由增加 `replan` 选项，图结构新增 `assistant → planner` 条件边。

### 依赖
- v0.8-1（Planner 支持 replan 模式）

### 改动文件

| 操作 | 文件 | 说明 |
|------|------|------|
| 修改 | `data/prompts/assistant.md` | Prompt 增加 replan 选项和触发条件 |
| 修改 | `server/orchestrator/graph.py` | 新增 assistant → planner 条件边 |

### 实现

**assistant.md** — 增加 replan 选项：

在 Decision Options 中增加：
```
- **replan**：调整学习计划。适用于：当前步骤反复学习仍无法突破，或 feedback 暴露前置知识欠缺需要先补基础
```

在 Constraints 中增加：
```
- 同一步骤学习 3 轮以上正确率持续低迷且 feedback.weaknesses 指向前置知识 → 倾向 "replan"
- 连续多步全对且剩余步骤内容高度相似 → 可考虑 "replan" 合并
- 不要频繁 replan（每步最多触发一次）
```

**graph.py** — 新增条件边：

```python
def decide_route(state: AgentState) -> str:
    action = state.get("next_action", "")
    if action == "done":
        return END
    if action == "replan":
        return "planner"
    return "resource"

def build_graph() -> StateGraph:
    # ... 现有节点不变 ...
    
    # assistant → resource / planner / END
    graph.add_conditional_edges("assistant", decide_route)
    
    # planner → assistant（重规划后回到路由）
    # 已有：planner → assistant
```

图结构更新为：
```
planner → assistant ──┬─ next/repeat → resource → practice → diagnose → assistant
            ↑         ├─ done → END                                  │
            │         └─ replan ─────────────────────────────────────┘
            └────────────────────────────────────────────────────────┘
```

### 测试
- 单元测试：decide_route("replan") → "planner"
- 集成测试：Assistant 在卡住场景下可能输出 replan

### 验收标准
- [ ] Assistant Prompt 含 replan 选项
- [ ] graph 包含 assistant → planner 边（replan 路径）

## v0.8-3 前端适配：计划动态更新 [ ]

### 目标
前端 PlanView 支持计划在流程中被动态更新（步骤增减时重新渲染）。

### 依赖
- v0.8-2（图结构调整完成）

### 改动文件

| 操作 | 文件 | 说明 |
|------|------|------|
| 修改 | `client/src/App.tsx` | PlanView 支持动态 plan；结果显示 planner 变更 |

### 实现

**PlanView 改动**：
- phase 为 "plan" 时渲染 plan 列表（已有）
- phase 为 "resource" 时，如果 plan 与上次不同，在页面上方显示「✏️ 学习计划已调整」提示条
- 不需要新增 phase，plan 更新是悄无声息的（前端一直从 taskState.plan 读取）

**ResultView 改动**：
- 当 next_action = "replan" 时，显示「🔄 正在调整学习计划...」
- 用户点击继续后，进入 resource 页时会看到更新后的 plan

```tsx
// ResultView 中增加 replan 分支
{next_action === "replan" ? (
  <>
    <p className="action-hint">学习计划需要调整</p>
    <button onClick={onContinue}>查看新计划</button>
  </>
) : ...}
```

### 测试
- 浏览器手动验证：触发 replan 后 plan 显示正确更新

### 验收标准
- [ ] plan 被 Planner 更新后前端正确渲染新步骤
- [ ] next_action = "replan" 时 ResultView 显示对应提示
