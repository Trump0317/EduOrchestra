### 实现

**AgentState** (`orchestrator/state.py`) — 数据结构放实际代码：

```python
from typing import TypedDict, Optional


class PlanStep(TypedDict):
    """计划中的一个学习步骤"""
    title: str       # 步骤标题，如“理解二次函数定义”
    desc: str        # 步骤描述，如“学习标准形式和图像特征”


class ResourceEntry(TypedDict):
    """一条学习资料"""
    type: str        # "video" | "article"
    title: str       # 资料标题
    url: str         # 资料链接


class Question(TypedDict):
    """一道练习题"""
    id: str          # 唯一标识，如 "q1"
    content: str     # 题目内容
    options: list[str]  # 选项列表，如 ["A. x=0", "B. x=1", ...]
    answer: str      # 正确答案，如 "A"
    kp: str          # 关联知识点，如 "二次函数顶点"


class Answer(TypedDict):
    """学生提交的一道答案"""
    question_id: str     # 对应 Question.id
    student_answer: str  # 学生选的选项，如 "A"
    is_correct: bool     # 判题结果
    correct_answer: str  # 正确答案（便于反馈时引用）


class AgentState(TypedDict):
    """LangGraph 共享状态。

    只有 feedback 节点能修改 current_step，其余节点只读。
    next_action 由 feedback 写入，decide_next 读取做路由。
    """
    task_id: str            # 任务唯一标识
    task_goal: str          # 学生设定的学习目标，如“掌握二次函数”
    plan: list[PlanStep]    # 学习步骤序列，planner 产出
    current_step: int       # 当前步骤索引（0-based），仅 feedback 修改
    resources: list[ResourceEntry]  # 当前步骤的学习资料，resource 产出
    questions: list[Question]       # 当前步骤的练习题，practice 产出
    waiting_for_answer: bool        # True = 等待学生提交答案，图已暂停
    answers: list[Answer]           # 学生提交的答案（含判题结果）
    analytics: Optional[dict]       # 学情分析，analytics 产出
    feedback: Optional[dict]        # 反馈报告，feedback 产出
    next_action: str                # "repeat_step" | "next_step" | ""
```

**5 个 Agent 节点**（`orchestrator/agents/`），均为纯函数 `fn(state) -> dict`，返回部分 state 更新：

- **planner**: 无视输入目标，固定返回两步计划。**不修改 current_step**（由 feedback 控制）
- **resource**: 取 `plan[current_step]`，返回两条固定 ResourceEntry（video + article）
- **practice**: 返回硬编码 3 道选择题（取前 2 道），设置 waiting_for_answer=True。额外导出 `check_answer(question, student_answer) -> bool` 纯函数
- **analytics**: 遍历 answers，统计 total / correct / accuracy
- **feedback**: 根据 accuracy 决定路由 + 步骤推进：
  - <0.6 → repeat_step（current_step 不变）
  - ≥0.6 → next_step + current_step += 1

**状态图** (`orchestrator/graph.py`)

```
build_graph():
  StateGraph(AgentState)
  + 5 个 node
  + 线性边: planner → resource → practice → analytics → feedback
  + 条件边: feedback → decide_next
      "repeat_step" → planner（重新计划同一 current_step）
      "next_step" + current_step < len(plan) → resource（进入下一步）
      "next_step" + current_step >= len(plan) → END（全部完成）
  + entry_point = "planner"
  + compile(interrupt_before=["practice"])
```

### 测试

测试文件：`server/tests/unit/test_graph_structure.py`，使用 `make_base_state(**overrides)` 辅助函数减少样板代码。

**Agent 节点测试（8 个）**：
- planner 返回固定 plan 含 title/desc
- resource 返回非空 resources
- practice 返回非空 questions + waiting=True
- check_answer 同答案→True，不同→False
- analytics 给定 1 对 1 错 → accuracy=0.5
- feedback accuracy=0 → next_action="repeat_step"
- feedback accuracy=1 → next_action="next_step"
- feedback accuracy=0.5 → next_action="repeat_step"

**图结构测试（4 个）**：
- 图包含全部 5 个节点
- planner→resource→practice 链路存在
- entry_point 为 planner
- "practice" 在 interrupt_before 中
