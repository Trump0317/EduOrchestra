# v0.9 — 记忆系统 + Tool Calling 基础设施

> 目标：建立 Tool Calling 基础设施，首批工具为 `save_memory` / `read_memory`，实现 LLM 自主读写的两层记忆系统。

## 架构变更

```
v0.8                                   v0.9
────                                   ────
assistant:                             assistant (bind_tools):
  render_prompt → llm_invoke_json        system_prompt + HumanMessage(context)
  → {"action": "..."}                    → LLM 思考
                                         ├─ tool_call: read_memory("weakness:")
                                         │    → 工具执行 → 结果回来
                                         ├─ tool_call: save_memory("...", "...")
                                         │    → 工具执行 → 结果回来
                                         └─ 最终: {"action": "...", "reason": "..."}
```

- Assistant 从 Prompt 文本升级为 `bind_tools` + 工具调用循环
- 两个 LangChain Tool：`save_memory` / `read_memory`
- 工具在 Assistant 节点内部闭环执行（不引入 ToolNode 到图中）
- 记忆存储：JSON 文件（零数据库依赖）

## 工具定义

### save_memory
```
名称: save_memory
参数: key (str), value (str)
返回: "记忆已保存: {key}"
用途: Assistant 发现学生的持续性薄弱点/强项时写入
```

### read_memory
```
名称: read_memory  
参数: prefix (str, 可选) — 前缀过滤，如 "weakness:"，空字符串返回全部
返回: 格式化记忆文本
用途: Assistant 决策前查看学生过去的表现记录
```

## 存储结构

```
data/
├── profile.json           ← 长期画像（任务完成后自动提炼）
├── memories.json          ← 键值记忆（Assistant 通过 tool 写入）
└── tasks/
    └── {task_id}.json     ← 已完成任务快照
```

## v0.9-1 记忆存储 + LangChain Tool 定义 [ ]

### 目标

### 依赖

### 改动文件

| 操作 | 文件 | 说明 |
|------|------|------|
| 新建 | `server/memory.py` | JSON 存储 + LangChain Tool 定义 |

### 实现

**memory.py**：

```python
"""记忆系统 — JSON 文件存储 + LangChain Tool。

存储结构:
  data/profile.json   — 长期画像
  data/memories.json  — 键值记忆 (Assistant tool 写入)
  data/tasks/{id}.json — 已完成任务快照
"""

import json
from pathlib import Path
from datetime import datetime
from langchain_core.tools import tool

DATA_DIR = Path("data")
TASKS_DIR = DATA_DIR / "tasks"
PROFILE_PATH = DATA_DIR / "profile.json"
MEMORIES_PATH = DATA_DIR / "memories.json"


def _ensure_dirs():
    TASKS_DIR.mkdir(parents=True, exist_ok=True)


def _read_json(path: Path, default=None):
    if default is None:
        default = {}
    if not path.exists():
        return default
    return json.loads(path.read_text(encoding="utf-8"))


def _write_json(path: Path, data):
    path.parent.mkdir(parents=True, exist_ok=True)
    path.write_text(
        json.dumps(data, ensure_ascii=False, indent=2), encoding="utf-8"
    )


# ── LangChain Tools ──

@tool
def save_memory(key: str, value: str) -> str:
    """写入一条长期记忆。用于记录学生持续性的学习特征。
    
    Args:
        key: 记忆键，建议格式 'weakness:<知识点>' 或 'strength:<知识点>' 或 'note:<描述>'
        value: 记忆内容，描述具体观察
    """
    _ensure_dirs()
    memories = _read_json(MEMORIES_PATH)
    memories[key] = value
    _write_json(MEMORIES_PATH, memories)
    return f"记忆已保存: {key}"


@tool
def read_memory(prefix: str = "") -> str:
    """读取长期记忆。用于决策前了解学生过去表现。
    
    Args:
        prefix: 前缀过滤，如 'weakness:' 只返回薄弱点。空字符串返回全部。
    """
    memories = _read_json(MEMORIES_PATH)
    if prefix:
        memories = {k: v for k, v in memories.items() if k.startswith(prefix)}
    if not memories:
        return "暂无相关记忆。"
    return "\n".join(f"- {k}: {v}" for k, v in memories.items())


# Assistant 绑定的工具列表
MEMORY_TOOLS = [save_memory, read_memory]


# ── 画像管理（任务完成后自动调用，非 Tool）──

def get_profile() -> dict:
    return _read_json(PROFILE_PATH, {"topics": {}, "skill_tags": {}, "stats": {}})


def archive_task(task_id: str, state: dict):
    _ensure_dirs()
    _write_json(TASKS_DIR / f"{task_id}.json", state)


def distill_from_task(task_id: str, state: dict):
    """任务完成后提炼长期画像。"""
    archive_task(task_id, state)

    task_goal = state.get("task_goal", "")
    topic = task_goal.replace("掌握", "").replace("学会", "").strip()[:20]
    step_history = state.get("step_history", [])
    best = max((h["best_accuracy"] for h in step_history), default=0.0)

    profile = get_profile()
    prev = profile["topics"].get(topic, {})
    profile["topics"][topic] = {
        "best_accuracy": max(best, prev.get("best_accuracy", 0)),
        "sessions": prev.get("sessions", 0) + 1,
        "last_at": datetime.now().isoformat(),
    }

    feedback = state.get("feedback", {})
    tags = profile.get("skill_tags", {})
    strengths = feedback.get("strengths", [])
    weaknesses = feedback.get("weaknesses", [])
    if strengths:
        tags["strengths"] = list(set(tags.get("strengths", []) + strengths))[:10]
    if weaknesses:
        tags["weaknesses"] = list(set(tags.get("weaknesses", []) + weaknesses))[:10]
    profile["skill_tags"] = tags

    stats = profile.get("stats", {})
    stats["total_tasks"] = stats.get("total_tasks", 0) + 1
    stats["completed_tasks"] = stats.get("completed_tasks", 0) + 1
    stats["total_sessions"] = stats.get("total_sessions", 0) + 1
    profile["stats"] = stats

    _write_json(PROFILE_PATH, profile)
```

## v0.9-2 Assistant 重构为 Tool Calling 模式 [ ]

### 目标
Assistant 从 `render_prompt → llm_invoke_json` 升级为 `bind_tools → tool_call 循环 → 决策`。

### 依赖
- v0.9-1

### 改动文件

| 操作 | 文件 | 说明 |
|------|------|------|
| 修改 | `server/orchestrator/agents/assistant.py` | Tool Calling 循环 |
| 修改 | `data/prompts/assistant.md` | System Prompt（含工具说明） |

### 实现

**assistant.py** — Tool Calling 循环：

核心流程：
```
1. 构建 messages = [SystemMessage(prompt), HumanMessage(context_json)]
2. llm_with_tools.invoke(messages)
3. 如果 response.tool_calls → 执行工具 → ToolMessage 追加到 messages → 回到 2
4. 如果 response.content（最终回答）→ 解析 JSON → 返回决策
```

```python
from langchain_core.messages import SystemMessage, HumanMessage, ToolMessage
from orchestrator.llm import get_llm
from memory import MEMORY_TOOLS, save_memory, read_memory

# 工具名 → 函数映射
_TOOL_MAP = {"save_memory": save_memory, "read_memory": read_memory}

def assistant_node(state: AgentState) -> dict:
    llm = get_llm().bind_tools(MEMORY_TOOLS)
    
    system = SystemMessage(content=render_prompt("assistant", ...context...))
    context = HumanMessage(content=_build_context_json(state))
    
    # 注入已有记忆供第一次决策参考
    memory_text = read_memory.invoke({"prefix": ""})
    context = HumanMessage(content=_build_context_json(state) + 
                          f"\n\n## 已有长期记忆\n{memory_text}")
    
    messages = [system, context]
    max_iterations = 3
    
    for _ in range(max_iterations):
        response = llm.invoke(messages)
        messages.append(response)
        
        if response.tool_calls:
            for tc in response.tool_calls:
                tool_func = _TOOL_MAP[tc["name"]]
                result = tool_func.invoke(tc["args"])
                messages.append(ToolMessage(content=result, tool_call_id=tc["id"]))
            continue  # 工具执行后 LLM 再决策
        else:
            decision = json.loads(response.content)
            return _build_update(state, decision)
    
    # 兜底
    return {"next_action": "next"}
```

**assistant.md** — 改为 System Prompt：

```
# Role
你是 EduOrchestra 的学习主管 AI。你可以使用工具查看和记录学生的长期记忆，然后做出路由决策。

# Available Tools
- read_memory(prefix): 读取长期记忆。prefix 如 "weakness:" 只返回薄弱点。空字符串返回全部。
- save_memory(key, value): 写入长期记忆。key 建议用 "weakness:<知识点>" 或 "strength:<知识点>"。

# When to Use Tools
- 决策前：先 read_memory 了解学生过去表现
- 发现模式：学生反复在同一类型出错 → save_memory 记录
- 无需工具：首次调用（无 feedback）直接决策

# Decision Options
- next / repeat / replan / done

# Output Format
最终回答必须是 JSON（不要 markdown 代码块）：
{"action": "next|repeat|replan|done", "reason": "决策理由（20字内）"}
```
