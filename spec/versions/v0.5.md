# v0.5 — Assistant 主管架构（MVP）

> 目标：Planner 提升为全局主管 Assistant，合并分析和反馈职责，前端拆分资源/做题页面。

## 架构变更

```
v0.3/v0.4                           v0.5
─────────                           ────
planner (建计划就退场)               assistant (全局主管，贯穿全程)
  ↓                                   ↓
resource ──→ practice(中断)          resource ──→ practice(中断)
  ↓                                   ↑              │
analytics (分析)                      │   resume     │
  ↓                                   │              ↓
feedback (反馈+路由) ──→ planner     assistant ──→ decide → resource or END
```

- Analytics / Feedback 节点删除，合并进 Assistant
- Assistant 每次调用都持有 `plan + step_history + answers + task_goal` 全量上下文
- 路由由 Assistant 的 `action` 字段决定

## v0.5-1 Assistant 节点 + State 重设计 [x]

### 目标
删除 analytics/feedback 节点，将 Planner 重命名为 Assistant，一个 Prompt 完成计划制定 + 答题分析 + 反馈生成 + 路由决策。

### 依赖
- v0.4（REST API + 前端骨架）

### 改动文件

| 操作 | 文件 | 说明 |
|------|------|------|
| 重命名 | `server/orchestrator/agents/planner.py → assistant.py` | Assistant 节点 |
| 删除 | `server/orchestrator/agents/analytics.py` | 合并进 Assistant |
| 删除 | `server/orchestrator/agents/feedback.py` | 合并进 Assistant |
| 修改 | `server/orchestrator/state.py` | 新增 step_history，去掉 analytics |
| 修改 | `data/prompts/planner.md → assistant.md` | 新 Prompt（两种模式） |
| 删除 | `data/prompts/analytics.md` | 不再需要 |
| 删除 | `data/prompts/feedback.md` | 不再需要 |

### 实现

**State 变更**：

- 新增 `step_history: list[StepRecord]` — 每个步骤的学习记录
- 去掉 `analytics: Optional[dict]` — 统计直接放 Assistant 输出
- `next_action` 值改为 `"repeat" | "next" | "done"`

**Assistant Prompt 设计**：

首次调用（无答题记录）：输出 plan + action="next"
答题后调用（有答题记录+step_history）：输出 feedback + action

Assistant 决策规则：
- 正确率 >= 0.6 → next 或 done（最后一步）
- 正确率 < 0.6 且已学 < 3 轮 → repeat
- 已学 >= 3 轮 → 强制 next

### 测试
更新 `test_graph_structure.py`：删 analytics/feedback 测试，改 planner 测试为 assistant。

### 验收标准
- [x] `assistant.py` 导入无报错
- [x] 首次调用返回 plan + action="next"
- [x] 答题后调用返回 feedback + 合理 action
- [x] step_history 正确记录每轮学习

## v0.5-3 前端分步页面 [x]

### 目标
将资源和做题拆为独立页面，用户先看资料再点"开始做题"。

### 依赖
- v0.5-2

### 改动文件

| 操作 | 文件 | 说明 |
|------|------|------|
| 修改 | `client/src/App.tsx` | 新增 resource/practice 独立页面 |
| 修改 | `client/src/App.css` | 新布局 |

### 实现

前端页面流：

```
idle → plan(看全计划) → resource(看资料+开始做题) → practice(做题) → result(助手反馈)
                         ↑                                                    │
                         │             repeat → practice                      │
                         │             next → resource                        │
                         └────────────────────────────────────────────────────┘
```

- Phase: "idle" | "loading" | "plan" | "resource" | "practice" | "result"
- 进度圆点指示器，显示当前步骤位置
- Resource 页面纯展示（资料列表 + 步骤标题）
- Practice 页面答题（radio group）

### 测试
手动验证：启动后端+前端，浏览器走通全流程，确认页面转换流畅。

### 验收标准
- [x] 输入目标 → 显示计划
- [x] 点击开始 → 显示资料页（不直接做题）
- [x] 点击开始做题 → 显示题目
- [x] 提交答案 → 显示助手反馈
- [x] repeat → 重新进入做题页
