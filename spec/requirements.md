# EduOrchestra 需求文档

> 状态：v0.5 MVP 完成 | 日期：2026-06-24

---

## 1. 项目定位

EduOrchestra 是一个**多智能体协作的模拟课堂**，面向 K12 学生自主学习场景。学生设定学习任务（目标），多个 AI Agent 协作完成：制定计划 → 搜集学习资料 → 出题练习 → 学情诊断 → 反馈报告，并根据反馈自动调整计划。

**核心理念**：AI 学习伙伴——既能在学生迷茫时给出方向，又能在学生有明确目标时辅助拆解执行。

---

## 2. 用户与场景

### 2.1 目标用户

| 维度 | 定位 |
|------|------|
| **学段** | MVP 聚焦 **高一数学** |
| **学科** | 先做数学，验证后扩展 |
| **核心使用者** | 学生（直接操作，自主使用） |

### 2.2 使用场景

| 场景 | 描述 |
|------|------|
| **课后自学巩固** | 学生放学后自主使用，查漏补缺 |
| **目标驱动学习** | 学生设定目标（如"掌握二次函数"），系统拆解任务 |

### 2.3 交互形态

- **结构化任务流**：创建任务 → 自动流转（计划 → 资源 → 做题 → 诊断 → 反馈 → 调整），做题节点暂停等待学生提交
- **对话式**：侧边常驻 Chat，学生随时提问，AI 携带当前任务上下文回答

---

## 3. 核心功能

### 3.1 全局主管（Assistant Agent）

- 学生设定学习目标（自然语言）→ 拆解为步骤序列
- 每轮答题后分析表现 → 生成反馈 → 决定路由
- 路由: repeat（重学当前步）/ next（进入下一步）/ done（全部完成）
- 持有 plan + step_history + answers 的全量上下文

### 3.2 资源搜集（Resource Agent）

- LLM 根据当前计划步骤搜索推荐学习资料
- 输出：视频链接、文章链接、知识点总结等

### 3.3 做题练习（Practice 节点）

- 从预置题库读取与当前步骤关联的题目
- 逐题展示，学生提交答案后实时判对错
- 记录答题结果（题号、学生答案、是否正确）

### 3.4 对话辅导（Chat）

- 独立于主流程，学生随时可用（v0.6+）
- LLM 对话，携带当前任务上下文

## 4. 多智能体架构

### 4.1 Agent 角色定义

| Agent | 职责 | LLM | 数据依赖 |
|-------|------|-----|---------|
| **Assistant** | 全局主管：目标拆解 + 答题分析 + 反馈生成 + 路由决策 | ✅ | task_goal, step_history, answers |
| **Resource** | 搜索推荐学习视频/资料 | ✅ | 当前步骤信息 |
| **Practice** | 从题库出题 + 判对错 | ❌ | 题库 |
| **Chat** | 对话辅导（独立） | ✅ | 当前任务上下文（v0.6+） |

### 4.2 编排方式

LangGraph 状态图管理流转，Assistant → Resource → Practice → Assistant 循环。Practice 节点暂停（interrupt）等待学生提交答案。

Chat Agent 不参与状态图，作为独立端点运行（v0.6+）。

### 4.3 状态管理

MVP 阶段使用 LangGraph MemorySaver（内存），暂不持久化。后续版本引入 SQLite checkpoint。

## 5. 技术方案

### 5.1 技术栈

| 层 | 技术选型 |
|----|---------|
| 后端语言 | Python 3.12+ |
| 多智能体框架 | LangGraph（状态图编排） |
| LLM | ChatOpenAI（OpenAI-compatible 协议，含 DeepSeek） |
| Web 框架 | FastAPI |
| 前端 | React 18 + Vite（当前版本先聚焦后端） |
| 题库 | JSON 文件（预置 5-10 道题） |

### 5.2 Prompt 管理

Prompt 模板存放在 `data/prompts/` 目录，与代码分离，便于调优。

### 5.3 架构概述

```
┌─ LangGraph 状态图（MemorySaver）──────────────┐
│                                                │
│  Assistant ←── Resource ──→ Practice(⏸)       │
│  (LLM 主管)    (LLM)        (题库+判题)        │
│                                                │
│  Assistant: 首次→定计划; 答题后→反馈+路由      │
│  Resource:  推荐学习资料                       │
│  Practice:  出题 + 判题 + 中断等答案           │
│                                                │
└────────────────────────────────────────────────┘
                   ↑ 共享上下文 (AgentState)
┌─ 侧边 ──────────┴─────────────────────────────┐
│  Chat (LLM, 独立端点, v0.6+)                   │
└────────────────────────────────────────────────┘
```

## 6. MVP 范围

### 6.1 目标

验证核心链路：**目标 → 计划 → 资源 → 做题 → 诊断 → 反馈 → 调整**

### 6.2 范围界定

| 维度 | MVP 范围 |
|------|---------|
| **学段** | 高一数学（先覆盖 1-2 个知识点） |
| **Agent** | 5 个 Agent + Chat，全部真实 LLM 驱动 |
| **题库** | 预置 5-10 道 JSON 题目 |
| **交互** | 做题节点暂停，学生提交答案后继续流转 |
| **前端** | 当前版本不开发，通过 API + 测试验证 |
| **状态** | 内存（MemorySaver），重启丢失 |
| **用户** | 单用户本地模式 |

### 6.3 不做什么（MVP）

- 不做 SQLite 持久化
- 不做 AI 题目生成
- 不做前端界面
- 不做知识图谱
- 不做用户认证
