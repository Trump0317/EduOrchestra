# EduOrchestra 需求文档

> 状态：初稿 | 日期：2026-06-22 | 基于需求访谈整理

---

## 1. 项目定位

EduOrchestra 是一个面向 **K12学生自主学习** 场景的多智能体系统，致力于解决：
- **资源搜集**：AI 生成 + 高考真题导入，构建结构化数学题库（习题、试卷），辅以教学视频链接
- **学情分析**：诊断学生知识薄弱点，构建知识图谱
- **反馈生成**：实时题目反馈 + 阶段性报告 + 异常预警
- **计划制定**：根据学生目标（如"搞懂二次函数"）拆解为可执行任务序列

**核心理念**：学生直接使用，系统充当 AI 学习伙伴——既能在学生迷茫时给出方向，又能在学生有明确目标时辅助拆解执行。

## 2. 用户与场景

### 2.1 目标用户

| 维度 | 定位 |
|------|------|
| **学段** | K12（中小学），MVP 聚焦 **高一数学** |
| **学科** | 先做 **数学**，验证后再扩展 |
| **核心使用者** | **学生**（直接操作，自主使用） |
| **次要使用者** | 家长/老师（后续可查看学情报告） |

### 2.2 使用场景

| 场景 | 描述 |
|------|------|
| **课后自学巩固** | 学生放学后自主使用，查漏补缺、做练习 |
| **备考冲刺** | 考试前集中复习、刷题、定位薄弱点 |
| **目标驱动学习** | 学生设定目标（如"掌握二次函数"），系统拆解任务 |

### 2.3 交互形态

**混合模式**：
- **结构化任务流**：系统推送每日任务 → 学生完成 → 分析反馈 → 下一轮
- **对话式**：学生随时可以对话提问，AI 老师讲解题目、答疑

## 3. 核心功能

### 3.1 资源搜集

| 维度 | 方案 |
|------|------|
| **资源来源** | **主力**：LLM 根据知识点生成题目；**补充**：历年高考数学真题（2016-2025） |
| **资源类型** | 结构化题目（含题干、答案、解析、知识点标签、难度）；教学视频/文章 → **存链接** |
| **触发方式** | 系统根据学生学习目标自动生成 + 推荐匹配题目 |
| **处理流程** | AI 生成/真题导入 → 结构化解析（核心字段提取）→ 知识点匹配 → 分类入库 |

> 详细数据来源策略见 [7.2 题库数据来源策略](#72-题库数据来源策略)。

### 3.2 学情分析

| 维度 | 方案 |
|------|------|
| **分析目标** | 知识点薄弱诊断 + 学习路径推荐 |
| **采集数据** | 答题记录（正确率、用时、错题分布）为主（后续可扩展行为数据） |
| **知识图谱** | AI 辅助生成知识点树 + 人工审核，表示前置依赖关系 |
| **输出** | 学生知识掌握热力图、薄弱点列表、推荐学习路径 |

### 3.3 反馈机制

| 层级 | 内容 | 频率 |
|------|------|------|
| **实时反馈** | 每道题对错判断 + 详细解析 + 关联知识点提醒 | 每次答题后 |
| **阶段报告** | 本周学习总结、进步趋势、薄弱点变化 | 每周 |
| **异常预警** | 连续错误、学习时长异常下降 | 实时 |

### 3.4 计划制定

| 维度 | 方案 |
|------|------|
| **模式** | **目标驱动**：学生设定学习目标 → AI 拆解为任务序列 |
| **任务粒度** | 每日任务（做哪些题、看哪些资料） |
| **自适应** | 根据诊断结果动态调整后续任务 |
| **执行确认** | AI 生成计划 → 学生确认后执行 |

## 4. 多智能体架构

### 4.1 Agent 角色定义

| Agent | 职责 |
|-------|------|
| **资源搜集 Agent** | 调用 LLM 生成题目、导入高考真题、结构化解析、知识点匹配入库 |
| **学情分析 Agent** | 分析答题数据，诊断薄弱知识点，更新知识图谱 |
| **反馈生成 Agent** | 根据分析结果生成题目解析、阶段报告、预警通知 |
| **计划制定 Agent** | 将学习目标拆解为每日任务序列 |
| **对话/辅导 Agent** | 与学生一对一对话，讲解题目、引导思考 |
| **编排协调 Agent (Orchestrator)** | 调度上述 Agent，管理整体学习流程 |

### 4.2 协作模式

| 阶段 | 方案 |
|------|------|
| **MVP** | **Orchestrator 统一调度**：按预定义流程依次调用各 Agent |
| **后续** | 考虑引入事件驱动或对话式协作提升灵活性 |

### 4.3 多智能体框架

**LangGraph**：状态图编排，生产级，学习曲线适中，LangChain 生态。

## 6. MVP 范围

### 6.1 目标

验证核心链路：**做题 → 诊断 → 反馈 → 计划 → 推荐资源**

### 6.2 范围界定

| 维度 | MVP 范围 |
|------|---------|
| **学段** | 高一数学全册（约 40-50 个核心知识点，涵盖集合与逻辑、函数、三角函数、平面向量、数列、不等式等） |
| **Agent** | 6 个 Agent 全部有简化版 |
| **资源** | 高一全册的习题+试卷，少量教学视频链接 |
| **交互** | 结构化答题 + 简单对话（单轮问答） |
| **前端** | React 单页面，含答题区、对话区、报告区 |
| **数据库** | SQLite |
| **用户认证** | 无（单用户本地模式） |
| **部署** | 单机开发环境可跑 |

### 6.3 不做什么（MVP）

- 不覆盖高一以外的年级
- 不做复杂的多轮对话（如苏格拉底式引导提问）
- 不做微信小程序/移动端
- 不做家长/老师管理端
- 不做复杂的用户权限系统

## 7. MVP 关键设计决策

### 7.1 学情数据采集方案

**标准版**：每条答题记录包含：

| 字段 | 说明 |
|------|------|
| 题目 ID | 关联题库 |
| 对/错 | 布尔值 |
| 答题用时 | 秒 |
| 错误类型 | 枚举：计算错误 / 概念不清 / 审题失误 / 其他 |
| 知识点标签 | 关联知识点 ID，可多个 |

### 7.2 题库数据来源策略

经调研，MVP 阶段采用 **混合策略**，按优先级排列：

#### 策略 A：AI 生成题目（首选，MVP 主力）

用 LLM 根据 人教A版 高一数学知识点直接生成题目。
- **优点**：无需爬虫、格式可控、与知识点精准对齐
- **缺点**：LLM 生成的题目质量和准确性需校验
- **工作量**：约需 1-2 天编写生成 + 校验 Pipeline

#### 策略 B：开源项目数据参考

以下 GitHub 项目提供了可直接参考的数据结构和处理方案：

| 项目 | 信息 |
|------|------|
| **[JudgePeach/math-question-bank](https://github.com/JudgePeach/math-question-bank)** | ⭐11 · MIT · Python + FastAPI + SQLite · 人教A版 · 支持 LaTeX 解析、AI 智能解题、OCR 识图、试卷导入拆解 · **强烈推荐作为题库建库方案参考** |
| **[Brilliantccc/math_teaching](https://github.com/Brilliantccc/math_teaching)** | ⭐5 · MIT · Flask + SQLite · 覆盖初一至高三 · 含练习模式、错题本、智能组卷 |
| **[ECNUQuizHub](https://github.com/MdFne/ECNUQuizHub)** | ⭐5 · React + TypeScript + MongoDB · 教师端题库管理 |

> MathBank 的数据库 Schema 和 AI 解析流程可直接参考，其 `database.py` 中的 Question 模型（含 content、question_type、category_chapter、category_knowledge、difficulty、answer_markdown 等字段）与我们的需求高度一致。

#### 策略 C：高考试卷原题（补充）

直接使用 **历年高考数学真题** 作为题库补充：
- **权威性**：高考真题是高中数学最高质量题目，命题严谨、知识点覆盖全面
- **可获取性**：历年真题公开透明，gzsxtk.com 已有「十年（2016-2025）高考数学真题分类汇编」可直接下载
- **与高一衔接**：高考真题中大量基础题和中档题涉及高一知识点（集合、函数、三角函数等），可按知识点筛选
- **处理流程**：下载真题 PDF → OCR 识别 → AI 按知识点分类拆解 → 筛选高一相关题目 → 导入题库

#### MVP 推荐路径

```
AI 生成题目（主力 70%）→ 高考真题补充（20%）→ 人工精选校验（10%）
```

先跑通 AI 生成链路，快速获得 100+ 道结构化题目，再引入高考真题中与高一相关的部分。

### 7.3 题目解析格式

AI 解析题目输出**核心字段**：

| 字段 | 说明 |
|------|------|
| 题目文本 | 原始题干 |
| 正确答案 | 答案文本/数值 |
| 解析 | 解题思路和步骤说明 |
| 关联知识点 | 知识点 ID 列表 |
| 难度等级 | 简单 / 中等 / 困难 |

### 7.4 用户认证

**单用户本地模式**：MVP 不做账号系统，浏览器直接使用，数据存本地 SQLite。

### 7.5 Agent 协议

**边写代码边定义**：在 LangGraph 中构建状态图时自然明确各 Agent 的输入/输出 Schema，不提前过度设计。
